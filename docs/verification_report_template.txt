2. Introduction
---------------
Noi dung nen viet:

- Muc tieu project la xay dung moi truong UVM de verify APB FIFO slave.
- DUT ho tro ghi/doc DATA, doc STATUS, doc COUNT, bao loi overflow/underflow/invalid/read-only write.
- Ket qua mong doi la simulation tu check bang scoreboard, khong can doc log thu cong.

Example paragraph:

This project verifies an APB FIFO slave using a UVM-based self-checking testbench. The verification environment checks normal FIFO operation, register behavior, boundary conditions, and error responses. The final regression was run on EDA Playground with Cadence Xcelium and completed with no UVM errors or scoreboard mismatches.

7. Checking Methodology
-----------------------
Nen nhan manh day la self-checking testbench:

- Scoreboard khong copy output DUT lam expected.
- Scoreboard co FIFO model doc lap expected_fifo[$].
- Khi write DATA hop le, model push_back(data).
- Khi read DATA hop le, model pop_front() va compare voi PRDATA.
- STATUS/COUNT duoc tinh tu model, khong lay tu DUT.
- Error scenarios phai co PSLVERR.

10. Notes About Simulator Messages
----------------------------------
Neu thay $unit_0x... trong Xcelium log, giai thich nhu sau:

- $unit_0x67f934e9 la compilation-unit scope do simulator tu sinh.
- No khong phai module DUT va khong phai loi.
- Top simulation that su van la top.

Neu thay #1step warning:

- Warning den tu clocking block skew default input #1step output #1step.
- #1step la SystemVerilog timing step hop le.
- Day khong phai functional failure.

11. Conclusion
--------------
Example conclusion:

The APB FIFO slave was verified using a UVM self-checking environment. Directed tests covered reset, normal FIFO operation, full/overflow, empty/underflow, pointer wrap-around, read-only register writes, and invalid address handling. Random mixed traffic was also included. The final Xcelium regression completed with UVM_ERROR=0, UVM_FATAL=0, and no scoreboard mismatches. Therefore, the DUT passed the defined verification plan.
