# APB FIFO Verification using UVM

This repository contains a SystemVerilog/UVM verification environment for an APB FIFO slave. The testbench is self-checking: stimulus is generated by UVM sequences, bus activity is sampled by a monitor, and an independent scoreboard model checks FIFO ordering, status/count values, and error responses.

The project also includes EDA Playground-ready files and text artifacts for verification planning, testcase traceability, and simulation evidence.

## Verified Features

- Reset/default FIFO state.
- APB write/read access to the DATA register.
- FIFO first-in-first-out ordering.
- STATUS register empty/full reporting.
- COUNT register occupancy reporting.
- Full FIFO and overflow error handling.
- Empty FIFO and underflow error handling.
- Pointer wrap-around behavior.
- Read-only register write protection.
- Invalid address handling.
- Random mixed APB traffic.
- Basic APB setup/access protocol sequencing.

## DUT Register Map

| Address | Register | Access | Expected Behavior |
| --- | --- | --- | --- |
| `0x0` | DATA | Read/write | Write pushes data into FIFO. Read pops oldest FIFO data. |
| `0x4` | STATUS | Read-only | Bit 0 reports empty, bit 1 reports full. |
| `0x8` | COUNT | Read-only | Reports current FIFO occupancy. |
| Other | Invalid | Invalid | Access returns `PSLVERR`. |

## Running Locally

Use a simulator with SystemVerilog and UVM support.

VCS example:

```sh
vcs -sverilog -ntb_opts uvm -f sim/filelist.f -top top -R
```

Xcelium example:

```sh
xrun -sv -uvm -f sim/filelist.f -top top +UVM_TESTNAME=apb_fifo_full_regression_test -R
```

To save a local result artifact:

```sh
xrun -sv -uvm -f sim/filelist.f -top top +UVM_TESTNAME=apb_fifo_full_regression_test -R | tee docs/local_xcelium_result.txt
```

## Running on EDA Playground

Use the paste-ready files instead of the multi-file repository layout:

1. Paste `eda_playground/edaplayground_design.sv` into the Design pane.
2. Paste `eda_playground/edaplayground_testbench.sv` into the Testbench pane.
3. Select a SystemVerilog simulator with UVM 1.2 support, for example Cadence Xcelium.
4. Set top module to `top` if required.
5. Run the simulation.

The EDA Playground version runs `apb_fifo_full_regression_test` by default.

The EDA version is functionally the same verification demo but flattened into two paste-ready files. The normal repo version remains the cleaner source-of-truth multi-file layout. See `docs/eda_vs_repo_layout.txt` for the full comparison.
